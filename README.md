## 📌 Tentang Program

**Penghijauan GitHub** adalah program kolaborasi nyata yang diinisiasi oleh HMPS Manajemen Informatika Politeknik Negeri Medan. Program ini mengajak seluruh mahasiswa MI untuk bersama-sama membangun sebuah web dari fondasi hingga siap didemokan di depan umum.

Tujuan utama program ini:
- Membiasakan mahasiswa dengan alur kerja kolaborasi berbasis Git & GitHub
- Menghasilkan karya nyata yang bisa dijadikan portofolio
- Menghidupkan akun GitHub peserta agar lebih bernilai untuk prospek kerja

## 🚀 Cara Mulai Berkontribusi

### 1. Fork Repository ini
Klik tombol **Fork** di pojok kanan atas halaman ini.

### 2. Clone ke komputer kamu
```bash
git clone https://github.com/USERNAME_KAMU/Penghijauan-github.git
cd Penghijauan-github
```

### 3. Buat branch baru
```bash
git checkout main
git pull origin main
git checkout -b feature/nama-fiturmu
```

### 4. Kerjakan perubahan kamu
Tambahkan fitur, style, atau perbaikan yang ingin kamu kontribusikan.

### 5. Commit & Push
```bash
git add .
git commit -m "feat: deskripsi singkat perubahanmu"
git push origin feature/nama-fiturmu
```

### 6. Buat Pull Request
Buka GitHub, klik **"Compare & pull request"**, isi deskripsi PR dengan jelas, lalu submit. Tunggu review dari maintainer!

> 📖 Baca [`CONTRIBUTING.md`](./CONTRIBUTING.md) untuk panduan lengkap aturan kontribusi.

## 🏆 Penilaian Top 3

Tiga kontributor terbaik akan dipilih berdasarkan:

1. **Jumlah Pull Request yang berhasil di-merge** *(bukan sekadar dibuat)*
2. **Kualitas kontribusi** — apakah fitur/style yang ditambahkan bermanfaat dan berfungsi dengan baik
3. **Konsistensi** — aktif berkontribusi secara merata selama program, bukan hanya di akhir

> 💡 Lebih baik 3 PR berkualitas daripada 20 PR berisi perubahan sepele!

## ❓ FAQ

**Q: Saya semester 1, apakah bisa ikut?**
A: Bisa! Program ini terbuka untuk semua semester. Tidak perlu jago dulu, yang penting mau belajar dan berkontribusi.

**Q: Apakah harus paham coding dulu?**
A: Tidak harus mahir, tapi setidaknya sudah mengikuti seminar GitHub yang diadakan sebelumnya. Kontribusi sekecil apapun tetap dihargai.

**Q: Apakah boleh berkontribusi lebih dari satu fitur?**
A: Sangat boleh! Bahkan dianjurkan. Semakin banyak kontribusi berkualitas, semakin besar peluang masuk Top 3.

**Q: Bagaimana kalau PR saya ditolak?**
A: Maintainer akan memberikan catatan revisi. Perbaiki sesuai masukan dan submit ulang — ini bagian dari proses belajar!

**Q: Apakah boleh mengerjakan bersama teman?**
A: Boleh diskusi bersama, tapi kontribusi (commit & PR) harus dari akun GitHub masing-masing agar tercatat dengan benar.
